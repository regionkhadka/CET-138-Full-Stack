⚡ JavaScript

The programming language of the web — making pages dynamic and interactive

JavaScript (JS) is the only programming language that runs natively in web browsers. It allows developers to manipulate the DOM, respond to user events, fetch data from APIs, validate forms, and build complex single-page applications. It also runs on the server with Node.js.

📄 1. DOM Manipulation

The DOM (Document Object Model) is a tree representation of the HTML page. JavaScript can select, read, modify, add, or remove any element in real time.

// Select an element const heading = document.getElementById('myHeading'); // Change content and style heading.textContent = 'Updated!'; heading.style.color = 'red'; // Create and append a new element const p = document.createElement('p'); p.textContent = 'New paragraph'; document.body.appendChild(p);

👉 2. Event Listeners

JavaScript responds to user interactions through event listeners. Events include click, keyup, mouseover, submit, and many more.

const btn = document.getElementById('myBtn'); btn.addEventListener('click', function() { alert('Button clicked!'); }); input.addEventListener('keyup', (e) => { display.textContent = e.target.value; });

📋 3. Arrays, Functions & the Todo App

This demonstrates arrays, functions, event handling, and dynamic DOM updates together — common patterns in real JS applications.

let todos = []; function addTodo(text) { todos.push({ text, done: false }); renderTodos(); } function renderTodos() { list.innerHTML = todos.map((t, i) => `<li>${t.text} <button onclick="remove(${i})">X</button></li>` ).join(''); }

    🌐 4. Fetch API — Calling External APIs

    The fetch() function is used to make HTTP requests to APIs. It returns a Promise, which is handled with .then() or async/await. This is fundamental to dynamic, data-driven web apps.

    // Using async/await to fetch a random joke async function getJoke() { const res = await fetch('https://official-joke-api.appspot.com/random_joke'); const data = await res.json(); console.log(data.setup, data.punchline); }

    💾 5. Local Storage

    Local Storage allows data to persist in the browser between sessions. It stores key-value pairs as strings and is useful for saving user preferences or draft content.

    // Save to local storage localStorage.setItem('name', 'Alice'); // Read from local storage const name = localStorage.getItem('name'); // 'Alice' // Remove an item localStorage.removeItem('name');
