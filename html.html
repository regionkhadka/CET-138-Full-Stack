📄 HTML

HyperText Markup Language — the structure and content of the web

HTML was the first thing I learned on this course and it forms the backbone of everything we build. Without HTML there is no structure — no headings, no forms, no tables. Every website we visit is built on HTML, even the most complex ones. In this section I've put together examples of the HTML concepts I've found most useful and interesting.

🔖 1. Semantic HTML Elements

Semantic elements clearly describe their meaning to both the browser and the developer. Using them improves accessibility and search engine optimisation.

<header> — Top of the page, contains nav/logo
<nav> — Navigation links
<main> — Main content area (unique per page)
<article> — Self-contained content (e.g. blog post)
<section> — Thematic grouping of content
<aside> — Sidebar or supplementary content
<footer> — Bottom of the page
Why it matters: Screen readers and search engines understand the page structure better. A <nav> tells assistive technology "this is navigation" — a generic <div> does not.

📝 2. HTML Forms — Working Example

Forms allow users to submit data. This example demonstrates various input types, labels, validation attributes, and the fieldset element for grouping.

<form action="/submit" method="post"> <label for="email">Email:</label> <input type="email" id="email" required> <button type="submit">Submit</button> </form>

📈 3. HTML Tables

Tables are used to display structured, tabular data. Proper use of <thead>, <tbody>, <th scope> improves both semantics and accessibility.

<table> <thead> <tr><th scope="col">Name</th></tr> </thead> <tbody> <tr><td>Alice</td></tr> </tbody> </table>

🎥 4. Media Elements

HTML5 introduced native support for audio and video without plugins. The <figure> and <figcaption> elements semantically wrap images with captions.

<figure> <img src="photo.jpg" alt="Description" width="400"> <figcaption>A descriptive caption</figcaption> </figure>

🔑 5. HTML5 Input Types & Validation

HTML5 added many new input types that provide built-in validation and mobile-friendly keyboards — no JavaScript needed.
